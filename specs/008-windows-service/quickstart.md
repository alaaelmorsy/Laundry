# Quickstart Validation: Windows Service Deployment

**Date**: 2026-06-13 | **Feature**: 008-windows-service

---

## المتطلبات قبل الاختبار

- Windows 10/11 (64-bit)
- MySQL مثبت ويعمل (يدوياً)
- `.env` مُعدَّل بصحيح
- `release/laundry-setup.exe` مبني من الكود الجديد
- حساب Administrator

---

## السيناريو 1: التثبيت الأول (Fresh Install)

**الهدف**: التحقق من تسجيل الـ Service وإنشاء الـ Shortcut

```
1. شغّل laundry-setup.exe بصلاحيات Administrator
2. أكمل خطوات الـ installer
3. بعد انتهاء التثبيت
```

**التحقق**:
- [ ] افتح `services.msc` → ابحث عن "مغسلة بلس" → يجب أن يكون الحالة: Running
- [ ] يوجد أيقونة "مغسلة بلس" على Desktop
- [ ] اضغط الأيقونة → يفتح المتصفح على `https://localhost:3443`
- [ ] لا تظهر أي نافذة سوداء عند الضغط على الأيقونة
- [ ] صفحة تسجيل الدخول تظهر بشكل صحيح

---

## السيناريو 2: التشغيل التلقائي بعد إعادة التشغيل

**الهدف**: التحقق من أن الـ Service يبدأ مع Windows

```
1. أعد تشغيل الكمبيوتر (Restart)
2. انتظر 60 ثانية بعد ظهور الـ Desktop
3. افتح المتصفح مباشرة
```

**التحقق**:
- [ ] `https://localhost:3443` متاح بدون تشغيل أي exe يدوياً
- [ ] `services.msc` → "مغسلة بلس" → Running
- [ ] لا رسالة خطأ في Windows Event Viewer

---

## السيناريو 3: التحديث التلقائي

**الهدف**: التحقق من آلية الإشعار والتحديث

```
1. رفع إصدار جديد على GitHub Releases (version أكبر من الحالي)
2. انتظر دورة الفحص (6 ساعات) أو أعِد تشغيل الـ Service
3. افتح واجهة البرنامج في المتصفح
```

**التحقق**:
- [ ] يظهر banner/إشعار بوجود تحديث في الواجهة
- [ ] يعرض رقم الإصدار الجديد وملاحظاته
- [ ] عند الضغط "تثبيت" → يبدأ التحميل
- [ ] بعد اكتمال التحميل → يشتغل الـ installer تلقائياً
- [ ] بعد انتهاء الـ installer → البرنامج يعود على الإصدار الجديد

---

## السيناريو 4: التحديث (Update) من إصدار قديم

**الهدف**: التحقق من سلوك الـ installer عند التحديث

```
1. شغّل installer جديد على جهاز فيه تثبيت قائم
2. أكمل خطوات التثبيت
```

**التحقق**:
- [ ] الـ installer يوقف الـ service القديم تلقائياً
- [ ] ينسخ الملفات بنجاح (لا "access denied")
- [ ] يعيد تشغيل الـ service بعد التثبيت
- [ ] `.env` الأصلي محفوظ (لم يُستبدل)
- [ ] البرنامج يعمل فور انتهاء الـ installer

---

## السيناريو 5: إعادة التشغيل عند الفشل (Auto-Restart)

**الهدف**: التحقق من سياسة إعادة التشغيل

```
1. افتح Task Manager → Find laundry-app.exe
2. End Task (أوقفه قسراً)
3. انتظر 10 ثوانٍ
```

**التحقق**:
- [ ] `laundry-app.exe` يعود للعمل خلال 10-15 ثانية
- [ ] `services.msc` → "مغسلة بلس" → Running مجدداً
- [ ] `https://localhost:3443` متاح بعد 30 ثانية

---

## السيناريو 6: انقطاع الإنترنت أثناء فحص التحديث

**الهدف**: التحقق من الصمود عند انقطاع الشبكة

```
1. افصل الإنترنت
2. أعِد تشغيل الـ Service (nssm restart LaundryPlusApp في CMD كـ Admin)
```

**التحقق**:
- [ ] البرنامج يعمل بشكل طبيعي بدون إنترنت
- [ ] لا رسالة خطأ تظهر للمستخدم بسبب فشل فحص التحديث
- [ ] في Event Log: رسالة تفيد بفشل الفحص (silent fail)

---

## أوامر مفيدة للتحقق (CMD كـ Administrator)

```cmd
:: حالة الـ Service
sc query LaundryPlusApp

:: إيقاف/تشغيل
nssm stop LaundryPlusApp
nssm start LaundryPlusApp

:: مسح الـ Service (عند الإلغاء أو الاختبار)
nssm remove LaundryPlusApp confirm

:: Windows Event Log للبرنامج
eventvwr.msc → Windows Logs → Application → Filter by "LaundryPlusApp"
```
