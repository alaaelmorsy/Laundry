# Implementation Plan: تغليف البرنامج كملف تنفيذي مستقل (pkg)

**Branch**: `005-pkg-packaging` | **Date**: 2026-06-12 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/005-pkg-packaging/spec.md`

## Summary

تغليف سيرفر Express المحلي في ملف `laundry.exe` واحد يتضمن Node.js 18، يُشغَّل على أجهزة العملاء بدون تثبيت Node.js. الحل يعتمد على `pkg` من Vercel مع مودول مركزي `server/paths.js` للفصل بين مسارات الملفات الثابتة (داخل exe) والمسارات القابلة للكتابة (بجانب exe). التوزيع عبر ملف `.zip` يحتوي الـ exe + أدوات تسجيل Windows Service.

## Technical Context

**Language/Version**: Node.js 18 (CommonJS), PowerShell 5.1

**Primary Dependencies**: `pkg` (devDependency للبناء فقط), `nssm` (توزيع خارجي للـ Windows Service)

**Storage**: لا تغيير في MySQL. ملفات `data/`, `ssl/`, `.env` تبقى على القرص بجانب الـ exe.

**Testing**: اختبار يدوي على جهاز بدون Node.js وفق `quickstart.md`.

**Target Platform**: Windows 10+ (64-bit)

**Project Type**: Desktop web app (local Express server + browser frontend)

**Performance Goals**: بدء السيرفر في أقل من 15 ثانية. حجم exe أقل من 100 MB.

**Constraints**: لا dependencies جديدة في runtime. `pkg` فقط كـ devDependency. لا تغيير في واجهة المستخدم أو قاعدة البيانات.

**Scale/Scope**: تثبيت واحد لكل مغسلة. لا تأثير على منطق الأعمال.

## Implementation Phases

### Phase A: مودول المسارات المركزي (`server/paths.js`)

ينشئ ثابتَين يُستخدمان في جميع الملفات:

```js
const path = require('path');
const isPkg = typeof process.pkg !== 'undefined';
const EXEC_DIR = path.dirname(process.execPath);

const APP_ROOT  = path.join(__dirname, '..');
const DATA_ROOT = isPkg ? EXEC_DIR : path.join(__dirname, '..');

module.exports = { APP_ROOT, DATA_ROOT, isPkg, EXEC_DIR };
```

- `APP_ROOT`: للملفات الثابتة (screens, assets, package.json داخل exe)
- `DATA_ROOT`: للملفات القابلة للكتابة (data/, ssl/, .env بجانب exe)
- في وضع dev: كلاهما نفس مجلد المشروع — لا تغيير في السلوك

### Phase B: تعديل `server/index.js`

```js
// أضف في أعلى الملف (قبل أي require آخر):
const { DATA_ROOT, APP_ROOT } = require('./paths');
require('dotenv').config({ path: require('path').join(DATA_ROOT, '.env') });

// استبدل:
const ROOT = path.join(__dirname, '..');
// بـ:
const ROOT = APP_ROOT;
```

### Phase C: تعديل `server/services/updateService.js`

```js
// استبدل:
const ROOT = path.join(__dirname, '..', '..');
const DATA_DIR = path.join(ROOT, 'data');
const BACKUP_DIR = path.join(ROOT, 'backup');

// بـ:
const { APP_ROOT: ROOT, DATA_ROOT } = require('../paths');
const DATA_DIR = path.join(DATA_ROOT, 'data');
const BACKUP_DIR = path.join(DATA_ROOT, 'backup');
```

كذلك: `path.join(ROOT, 'scripts', 'updater.ps1')` → `path.join(DATA_ROOT, 'scripts', 'updater.ps1')`

### Phase D: تعديل `server/services/whatsappService.js`

```js
// استبدل:
const SESSION_DIR = path.join(__dirname, '../../data/whatsapp_session');
// بـ:
const { DATA_ROOT } = require('../paths');
const SESSION_DIR = path.join(DATA_ROOT, 'data', 'whatsapp_session');
```

### Phase G: إعداد pkg في `package.json`

```json
{
  "bin": "server/index.js",
  "scripts": {
    "build": "pkg . --compress GZip"
  },
  "pkg": {
    "targets": ["node18-win-x64"],
    "assets": [
      "screens/**/*",
      "assets/**/*",
      "node_modules/@whiskeysockets/baileys/WAProto/**"
    ],
    "outputPath": "dist"
  },
  "devDependencies": {
    "pkg": "^5.8.1"
  }
}
```

**ملاحظة Baileys**: مكتبة `@whiskeysockets/baileys` تستخدم ملفات `.proto` — يجب إدراجها كـ assets صريحة.

### Phase H: سكريبت `scripts/install-service.ps1`

```powershell
param(
  [string]$ExePath   = (Join-Path $PSScriptRoot "..\laundry.exe"),
  [string]$ServiceName = "LaundryPOS",
  [string]$NssmPath  = (Join-Path $PSScriptRoot "..\nssm.exe")
)

if (-not (Test-Path $NssmPath)) {
  Write-Host "❌ لم يُعثر على nssm.exe في: $NssmPath"
  exit 1
}
if (-not (Test-Path $ExePath)) {
  Write-Host "❌ لم يُعثر على laundry.exe في: $ExePath"
  exit 1
}

& $NssmPath install $ServiceName $ExePath
& $NssmPath set $ServiceName AppRestartDelay 5000
& $NssmPath set $ServiceName Description "PLUS Laundry POS System"
& $NssmPath start $ServiceName

Write-Host "✅ تم تسجيل خدمة $ServiceName بنجاح"
Write-Host "البرنامج سيبدأ تلقائياً مع Windows"
```

## Risk & Mitigations

| الخطر | الحل |
|-------|------|
| Baileys native bindings لا تعمل داخل exe | اختبار مبكر في Phase G؛ إذا فشل → تحميل Baileys من مسار DATA_ROOT خارج snapshot |
| `express.static` لا يقرأ من snapshot | pkg يدعم serving من snapshot افتراضياً؛ اختبار سريع يؤكد |
| حجم exe يتجاوز 100 MB | `--compress GZip` يقلص الحجم ~30%؛ استبعاد devDependencies تلقائي |
| مسارات تحتوي مسافات أو أحرف عربية | اختبار على `C:\برامج\laundry\` قبل الإصدار |
| `updater.ps1` لا يجد مسار exe الجديد بعد التحديث | نظام التحديث (spec 004) يحفظ الـ exe في نفس مجلد DATA_ROOT |
