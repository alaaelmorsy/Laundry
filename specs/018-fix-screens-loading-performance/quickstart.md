# Quickstart / Validation Guide

## الهدف
التحقق من أن شاشتي الخدمات والمنتجات تُحمَّلان بسرعة بعد التعديل، مع بقاء البيانات صحيحة.

## المتطلبات
- تشغيل التطبيق محلياً (`node server/server.js` أو عبر `laundry-app.exe`)
- وجود بيانات في قاعدة البيانات (خدمات + منتجات مع أسعار)

## سيناريوهات التحقق

### 1. فتح شاشة المنتجات
1. افتح الشاشة الرئيسية → اضغط "المنتجات"
2. **المتوقع**: يظهر جدول المنتجات خلال أقل من ثانيتين
3. **تحقق**: عمود "عدد العمليات" يعرض الأرقام الصحيحة لكل منتج
4. **تحقق**: الترتيب الافتراضي هو `sort_order ASC`

### 2. فتح شاشة الخدمات
1. اضغط "الخدمات" من القائمة الرئيسية
2. **المتوقع**: تظهر قائمة الخدمات خلال أقل من ثانية

### 3. الفرز بعمود "عدد العمليات"
1. في شاشة المنتجات، اضغط رأس عمود "عدد العمليات"
2. **المتوقع**: يُعاد ترتيب الجدول خلال ثانيتين
3. **تحقق**: المنتجات بعدد عمليات أكبر تظهر أولاً (أو آخراً حسب اتجاه الفرز)

### 4. البحث في المنتجات
1. اكتب اسم منتج في خانة البحث
2. **المتوقع**: تظهر النتائج بعد 300ms

### 5. صحة البيانات عند فتح نافذة تعديل منتج
1. اضغط "تعديل" على أي منتج
2. **المتوقع**: تظهر كل أسعار الخدمات الصحيحة للمنتج

## التحقق من الفهارس (اختياري)
```sql
SHOW INDEX FROM products;
SHOW INDEX FROM product_price_lines;
SHOW INDEX FROM laundry_services;
```
يجب أن تظهر:
- `idx_products_sort_order`
- `idx_products_search`
- `idx_ppl_product_id`
- `idx_services_sort_order`
- `idx_services_search`

## علامات النجاح
- [ ] شاشة المنتجات تفتح بسرعة ملحوظة مقارنة بما قبل
- [ ] شاشة الخدمات تفتح بسرعة ملحوظة
- [ ] عدد العمليات لكل منتج صحيح
- [ ] الفرز والبحث يعملان كالمعتاد
- [ ] لا توجد أخطاء في console عند فتح الشاشتين
