# Quickstart Validation: SuperAdmin Hidden User

## المتطلبات

- البرنامج يعمل (Node.js + MySQL)
- أي مستخدم `admin` عادي موجود في النظام

## سيناريوهات التحقق

### 1. تسجيل دخول superAdmin

1. افتح شاشة تسجيل الدخول
2. أدخل اسم المستخدم: `superAdmin` وكلمة المرور: `LearnTech`
3. **النتيجة المتوقعة**: يدخل البرنامج بنجاح

### 2. التحقق من الصلاحيات الكاملة

1. بعد دخول superAdmin، افتح أي شاشة محمية (مثل المستخدمين، الأدوار، الإعدادات)
2. **النتيجة المتوقعة**: جميع الشاشات والأزرار متاحة

### 3. إخفاء superAdmin من شاشة المستخدمين

1. سجّل دخول بأي مستخدم عادي أو أدمن
2. افتح شاشة المستخدمين
3. **النتيجة المتوقعة**: لا يظهر `superAdmin` في القائمة
4. ابحث عن `superAdmin`
5. **النتيجة المتوقعة**: لا توجد نتائج

### 4. حصرية استعادة النظام — أدمن عادي

1. سجّل دخول بمستخدم `admin` عادي
2. افتح الإعدادات
3. **النتيجة المتوقعة**: تبويب "استعادة النظام" غير مرئي في الشريط الجانبي

### 5. حصرية استعادة النظام — superAdmin

1. سجّل دخول بـ `superAdmin`
2. افتح الإعدادات
3. **النتيجة المتوقعة**: تبويب "استعادة النظام" مرئي ويعمل بشكل طبيعي

### 6. التحقق من حماية الخادم

1. سجّل دخول بمستخدم عادي
2. من المتصفح (DevTools → Console)، نفّذ:
   ```javascript
   fetch('/api/invoke', {
     method: 'POST',
     headers: {'Content-Type': 'application/json'},
     credentials: 'include',
     body: JSON.stringify({ method: 'systemRestore', payload: {} })
   }).then(r => r.json()).then(console.log)
   ```
3. **النتيجة المتوقعة**: `{ success: false, message: 'غير مصرح' }`

### 7. منع إنشاء مستخدم باسم superAdmin

1. سجّل دخول بأدمن، افتح شاشة المستخدمين
2. حاول إنشاء مستخدم جديد باسم `superAdmin`
3. **النتيجة المتوقعة**: رسالة خطأ ورفض الإنشاء
