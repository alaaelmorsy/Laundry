# Quickstart & Validation Guide: Global Sidebar Navigation

## المتطلبات الأساسية

- البرنامج يعمل (`npm start` أو الـ service)
- مستخدم admin وآخر cashier متاحان للاختبار

## سيناريوهات التحقق

### 1. ظهور القائمة على الكمبيوتر

1. سجّل دخولك كـ admin
2. افتح أي شاشة (مثلاً: الفواتير `/screens/invoices/invoices.html`)
3. **المتوقع**: القائمة الجانبية تظهر على اليمين (RTL) بعرض ~248px
4. **المتوقع**: رابط "الفواتير" مميَّز بلون مختلف (`.active`)
5. **المتوقع**: المحتوى الرئيسي للفواتير لا يتداخل مع القائمة

### 2. اختبار التنقل

1. من شاشة الفواتير، اضغط على "التقارير" في القائمة الجانبية
2. **المتوقع**: الانتقال إلى `/screens/reports/reports.html`
3. **المتوقع**: رابط "التقارير" أصبح `.active` والفواتير لم تعد `.active`

### 3. اختبار الطي/التوسيع

1. اضغط على زر الطي (← أو →) في رأس القائمة الجانبية
2. **المتوقع**: القائمة تتضيق إلى ~64px وتظهر الأيقونات فقط
3. **المتوقع**: المحتوى يتوسع لملء المساحة
4. انتقل إلى شاشة أخرى ثم ارجع
5. **المتوقع**: القائمة لا تزال مطوية (الحالة محفوظة في localStorage)

### 4. اختبار الجوال (Breakpoint)

1. افتح أدوات المطوّر (F12) → وضع الجوال أو عرض < 768px
2. **المتوقع**: القائمة الجانبية تختفي تماماً
3. **المتوقع**: لا مسافة فارغة على الجانب
4. **المتوقع**: المحتوى يملأ الشاشة بالكامل

### 5. اختبار الصلاحيات (Cashier)

1. سجّل دخولك كـ cashier (دور محدود الصلاحيات)
2. **المتوقع**: الروابط المقيدة (مثل: المستخدمون، الصلاحيات) مخفية من القائمة
3. **المتوقع**: الروابط المسموح بها تظهر بشكل طبيعي

## Regression Checks

- [ ] شاشة الإعدادات: التبويبات الداخلية (Laundry, Tax, Printer...) لا تزال تعمل
- [ ] شاشة POS: الـ layout الداخلي (عربة التسوق، قائمة المنتجات) لا يتداخل
- [ ] وضع اللغة الإنجليزية (LTR): القائمة تظهر على الجانب الصحيح
