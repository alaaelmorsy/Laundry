# Feature Specification: Full Self-Contained Installer Bundle

**Feature Branch**: `007-full-installer-bundle`

**Created**: 2026-06-13

**Status**: Draft

**Input**: User description: "اريد ان تدمج كل شئ وتشغله المطلوب وانا اثبت ملف exe واريد كل شئ يكون محفوظ فى المسار اللى هختاره اثناء التثبيت بحيث فقط المستخدم يثبت الماى سيكوال ويرفع قاعدة البيانات ويثبت البرنامج والبرنامج يشتغل تلقائى بدون اى تدخل اخر"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - تثبيت البرنامج لأول مرة (Priority: P1)

مدير المغسلة (أو الفني المسؤول) عنده جهاز نظيف عليه MySQL مثبّت وقاعدة البيانات مرفوعة. يشغّل `PLUS-Laundry-Setup.exe` ويختار مجلد التثبيت ثم ينتظر. بعد انتهاء التثبيت يجد المتصفح فتح تلقائياً على واجهة البرنامج دون أي خطوة إضافية.

**Why this priority**: هذا هو المسار الأساسي الوحيد — بدونه لا توجد قيمة للـ installer.

**Independent Test**: يُختبر بتشغيل الـ installer على جهاز به MySQL فقط، والتحقق من أن المتصفح يفتح على `https://localhost:3443` مع شاشة تسجيل الدخول خلال 60 ثانية من انتهاء التثبيت.

**Acceptance Scenarios**:

1. **Given** جهاز Windows 10/11 به MySQL مثبّت وقاعدة البيانات `laundry` موجودة، **When** يشغّل المستخدم `PLUS-Laundry-Setup.exe` ويكمل الخطوات، **Then** يُثبَّت البرنامج في المجلد المختار ويشتغل تلقائياً ويفتح المتصفح.
2. **Given** تثبيت مكتمل، **When** يُعاد تشغيل الجهاز، **Then** يبدأ البرنامج تلقائياً مع Windows بدون تدخل المستخدم.
3. **Given** المستخدم اختار مجلد تثبيت غير افتراضي (مثل `D:\MyLaundry`)، **Then** تُحفظ جميع ملفات البرنامج والبيانات في هذا المجلد تحديداً.

---

### User Story 2 - تحديث البرنامج (إعادة التثبيت) (Priority: P2)

عند صدور إصدار جديد، يُشغّل الفني نفس ملف الـ installer الجديد على جهاز به نسخة قديمة.

**Why this priority**: ضروري لاستمرارية الخدمة — إصدارات مستقبلية ستحتاجه.

**Independent Test**: تثبيت نسخة قديمة ثم تثبيت نسخة جديدة فوقها والتحقق من أن البيانات (`data/`) لم تُحذف وأن الإصدار الجديد يعمل.

**Acceptance Scenarios**:

1. **Given** نسخة قديمة مثبّتة وبها بيانات في `data/`، **When** يُشغَّل installer نسخة جديدة، **Then** تُستبدل ملفات البرنامج (`laundry-app.exe`) وتُحفظ بيانات المستخدم (`data/`) كما هي.
2. **Given** إعادة التثبيت مكتملة، **Then** يعمل البرنامج بالإصدار الجديد مع نفس البيانات السابقة.

---

### User Story 3 - معالجة بيئة Node.js المحذوفة (Priority: P2)

جهاز كان عليه Node.js ثم حُذف — تبقّت متغيرات بيئة قديمة (`OPENSSL_CONF`، `NODE_PATH`، إلخ) في Windows registry تُعيق تشغيل البرنامج.

**Why this priority**: مشكلة موثّقة تمنع تشغيل البرنامج على أجهزة تطوير سابقة.

**Independent Test**: على جهاز كان عليه Node.js مُحذَف، تثبيت البرنامج والتحقق من أنه يشتغل بشكل طبيعي بدون أخطاء OPENSSL أو crash.

**Acceptance Scenarios**:

1. **Given** جهاز به متغيرات بيئة Node.js متبقية في registry، **When** يُثبَّت البرنامج ويُشغَّل، **Then** يعمل بشكل طبيعي دون أي أخطاء مرتبطة بـ OpenSSL أو Node.
2. **Given** البرنامج مثبَّت، **Then** تُحذف متغيرات Node.js الضارة (`OPENSSL_CONF`, `OPENSSL_ENGINES`, `NODE_OPTIONS`, `NODE_PATH`) من System environment نهائياً أثناء التثبيت.

---

### Edge Cases

- ماذا يحدث لو كان الـ port 3443 مشغولاً ببرنامج آخر؟ → يُعرض تحذير أو يُوقف البرنامج القديم.
- ماذا لو لم يكن MySQL مثبتاً؟ → يُعرض رسالة واضحة تطلب من المستخدم تثبيته أولاً.
- ماذا لو كانت قاعدة البيانات `laundry` غير موجودة؟ → البرنامج يُنشئها تلقائياً عند أول تشغيل (السلوك الحالي في `db.initialize()`).
- ماذا لو شغّل المستخدم الـ `laundry-app.exe` مباشرة بدون `launcher.vbs`؟ → يجب أن يعمل أيضاً (متغيرات Node.js محذوفة من registry نهائياً).
- ماذا لو كان الـ installer مُشغَّلاً من مجلد به مسافات في اسمه؟ → يعمل بشكل صحيح.
- ماذا لو أُلغي التثبيت في منتصفه؟ → لا تبقى ملفات ناقصة أو خدمات مسجّلة.

---

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: الـ installer يجب أن يُتيح للمستخدم اختيار مجلد التثبيت بشكل حر (مع قيمة افتراضية مناسبة).
- **FR-002**: جميع ملفات البرنامج (`laundry-app.exe`, `mkcert.exe`, `launcher.vbs`, `scripts/`, `data/`, `ssl/`, `.env`) يجب أن تُنسخ في المجلد المختار فقط — لا شيء في مجلد آخر.
- **FR-003**: الـ installer يجب أن يُنظّف متغيرات بيئة Node.js الضارة (`OPENSSL_CONF`, `OPENSSL_ENGINES`, `NODE_OPTIONS`, `NODE_PATH`, `NODE_ENV`) من System environment في registry أثناء التثبيت.
- **FR-004**: الـ installer يجب أن يُثبّت شهادة mkcert CA ويولّد شهادات SSL خاصة بالجهاز.
- **FR-005**: الـ installer يجب أن يُسجّل البرنامج في Task Scheduler ليشتغل تلقائياً مع بداية Windows.
- **FR-006**: الـ installer يجب أن يشغّل البرنامج مباشرة عند انتهاء التثبيت ويفتح المتصفح على `https://localhost:3443`.
- **FR-007**: عند إعادة التثبيت، يجب أن تُحفظ بيانات `data/` كما هي بدون حذف.
- **FR-008**: الـ installer يجب أن يُوقف أي نسخة سابقة من البرنامج تعمل قبل نسخ الملفات الجديدة.
- **FR-009**: الـ installer يجب أن يشتغل بصلاحيات Administrator (UAC prompt تلقائي).
- **FR-010**: ملف `.env` يجب أن يُنسخ كـ Hidden file وأن يحتوي بيانات الاتصال بـ MySQL الصحيحة.
- **FR-011**: الـ `laundry-app.exe` عند تشغيله مباشرة (بدون launcher) يجب أن يعمل بشكل طبيعي بعد تنظيف متغيرات registry.
- **FR-012**: واجهة الـ installer باللغة العربية مع دعم RTL.

### Key Entities

- **Installer (`PLUS-Laundry-Setup.exe`)**: الملف الموزَّع للعملاء — يحتوي جميع ملفات البرنامج مضغوطة.
- **Application Bundle (`laundry-app.exe`)**: البرنامج الرئيسي المُغلَّف بـ pkg — يحتوي Node.js runtime داخله.
- **Installation Directory**: المجلد الذي يختاره المستخدم — مرجع واحد لكل الملفات.
- **System Environment Registry**: مصدر المتغيرات الضارة المتبقية من Node.js — يُنظَّف أثناء التثبيت.
- **Task Scheduler Entry (LaundryPOS)**: المسؤول عن التشغيل التلقائي مع Windows.
- **mkcert CA**: شهادة الجذر المثبَّتة في Windows trust store — تُمكّن HTTPS الموثوق.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: التثبيت يكتمل من الضغط على next حتى فتح المتصفح في أقل من 90 ثانية.
- **SC-002**: حجم ملف الـ installer أقل من 130 MB.
- **SC-003**: 100% من أجهزة العملاء الجديدة تفتح البرنامج بعد التثبيت مباشرة بدون تدخل إضافي.
- **SC-004**: البرنامج يبدأ تلقائياً مع Windows في 100% من حالات إعادة التشغيل.
- **SC-005**: أجهزة كانت عليها Node.js مُحذَف تعمل بشكل طبيعي بعد التثبيت (0 أخطاء OPENSSL).
- **SC-006**: بيانات المستخدم (`data/`) تُحفظ بالكامل عند إعادة التثبيت (0 فقدان بيانات).
- **SC-007**: جميع ملفات البرنامج محفوظة في المجلد الذي اختاره المستخدم أثناء التثبيت (0 ملفات خارجه).

## Assumptions

- MySQL مُثبَّت مسبقاً على جهاز العميل (هذا خارج نطاق الـ installer في المرحلة الحالية).
- قاعدة البيانات إما موجودة أو سيُنشئها البرنامج تلقائياً عند أول تشغيل.
- جهاز العميل يعمل بـ Windows 10 أو 11 (64-bit).
- المستخدم لديه صلاحية تثبيت برامج (يقبل UAC prompt).
- Chrome أو Edge مثبَّت على الجهاز لفتح واجهة البرنامج.
- بيانات اتصال MySQL (`host`, `user`, `password`, `port`) ثابتة ومعروفة مسبقاً عند البناء — أو يُتاح تخصيصها في شاشة الإعداد داخل الـ installer.
- `launcher.vbs` هو الوسيلة الوحيدة لتشغيل البرنامج بدون نافذة سوداء — Task Scheduler يستخدمه.
- الـ installer يُبنى من مجلد `release/` الذي يحتوي نواتج `npm run build`.
