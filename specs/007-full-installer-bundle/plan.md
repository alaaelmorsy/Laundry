# Implementation Plan: Full Self-Contained Installer Bundle

**Branch**: `007-full-installer-bundle` | **Date**: 2026-06-13 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/007-full-installer-bundle/spec.md`

## Summary

تحسين `installer/laundry.iss` ليحل ثلاث مشاكل جذرية: (1) حذف متغيرات Node.js الضارة من Windows System registry نهائياً أثناء التثبيت، (2) إيقاف النسخة القديمة من البرنامج قبل نسخ الملفات، (3) الحفاظ على `.env` عند التحديث. جميع الملفات تُحفظ في المجلد الذي يختاره المستخدم بدون أي تغيير في هيكل الـ release.

## Implementation Phases

### Phase A: اضافة [Registry] Section — حذف متغيرات Node.js

**الملف**: `installer/laundry.iss`

اضافة قسم `[Registry]` يحذف المتغيرات الضارة المتبقية من Node.js:

```pascal
[Registry]
Root: HKLM; Subkey: "SYSTEM\CurrentControlSet\Control\Session Manager\Environment"; \
  ValueName: "OPENSSL_CONF";    Flags: deletevalue; Tasks: ;
Root: HKLM; Subkey: "SYSTEM\CurrentControlSet\Control\Session Manager\Environment"; \
  ValueName: "OPENSSL_ENGINES"; Flags: deletevalue; Tasks: ;
Root: HKLM; Subkey: "SYSTEM\CurrentControlSet\Control\Session Manager\Environment"; \
  ValueName: "NODE_OPTIONS";    Flags: deletevalue; Tasks: ;
Root: HKLM; Subkey: "SYSTEM\CurrentControlSet\Control\Session Manager\Environment"; \
  ValueName: "NODE_PATH";       Flags: deletevalue; Tasks: ;
Root: HKLM; Subkey: "SYSTEM\CurrentControlSet\Control\Session Manager\Environment"; \
  ValueName: "NODE_ENV";        Flags: deletevalue; Tasks: ;
```

`deletevalue` flag آمن: إذا لم يكن الـ key موجوداً، Inno Setup يتجاهله بصمت.

### Phase B: اضافة [Code] Section — إيقاف البرنامج + Broadcast

```pascal
[Code]
procedure CurStepChanged(CurStep: TSetupStep);
var ResultCode: Integer;
begin
  if CurStep = ssInstall then begin
    Exec(ExpandConstant('{sys}\taskkill.exe'), '/F /IM laundry-app.exe', '',
         SW_HIDE, ewWaitUntilTerminated, ResultCode);
    Sleep(1500);
    SendBroadcastMessage($001A, 0, 'Environment');
  end;
end;
```

- `taskkill`: يوقف البرنامج القديم لتجنب "access denied" عند نسخ `laundry-app.exe`
- `SendBroadcastMessage($001A)` = `WM_WININICHANGE`: يُعلم Windows بتغيير متغيرات البيئة

### Phase C: حماية .env عند التحديث

تعديل entry `.env` في `[Files]` لاضافة `onlyifdoesntexist`:

```pascal
Source: "..\installer\.env"; DestDir: "{app}"; Attribs: hidden;
  Flags: ignoreversion onlyifdoesntexist uninsneveruninstall
```

قبل التعديل كان: `Flags: ignoreversion` فقط — كان يستبدل `.env` عند كل تحديث.

### Phase D: تحسين فتح المتصفح (اختياري — P3)

استبدال `ping` delay بـ `timeout` أوضح، مع `postinstall` flag:

```pascal
Filename: "{cmd}"; Parameters: "/c timeout /t 15 /nobreak >nul & start https://localhost:3443";
  Flags: runhidden postinstall nowait; Description: "فتح البرنامج في المتصفح"
```

## Risk & Mitigations

| الخطر | الحل |
|-------|------|
| deletevalue على key غير موجود | Inno Setup يتجاهله بصمت — آمن |
| taskkill على process غير موجود | ResultCode يتجاهل — لا خطأ |
| .env القديم لا يحتوي متغيرات جديدة عند التحديث | نادر — يُوثَّق في release notes |
