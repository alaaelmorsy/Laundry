# Quickstart: التحقق من نظام التحديث

**Date**: 2026-06-15

---

## المتطلبات

- تطبيق الغسيل يعمل على `localhost:3000`
- `github.owner` و `github.repo` مضبوطان في `package.json`
- إصدار جديد منشور على GitHub Releases (`laundry-app-v{X.Y.Z}.exe` + `sha256sums.txt`)

---

## السيناريو 1: فحص التحديث من الإعدادات

1. افتح المتصفح على `https://localhost:3000/screens/settings/settings.html`
2. انقر تبويب "التحديثات"
3. اضغط "فحص التحديثات"
4. **النتيجة المتوقعة**:
   - إذا يوجد إصدار جديد: يظهر رقم الإصدار الجديد وزر "تحديث الآن"
   - إذا لا يوجد تحديث: يظهر "✅ أنت تستخدم أحدث إصدار"

---

## السيناريو 2: التحقق من فحص صلاحية الدعم الفني

### إعداد: تعيين `support_expiry_date` كتاريخ ماضٍ

```sql
UPDATE app_settings SET support_expiry_date = '2020-01-01' WHERE id = 1;
```

1. تأكد أن هناك تحديث متاح (يوجد إصدار جديد على GitHub)
2. افتح الإعدادات → تبويب التحديثات
3. اضغط "فحص التحديثات" — **يجب أن ينجح**
4. اضغط "تحديث الآن"
5. **النتيجة المتوقعة**: رسالة "انتهت فترة الدعم الفني — يرجى تجديد الدعم للحصول على التحديثات"، لا يبدأ التحميل

### إعادة الوضع الطبيعي

```sql
UPDATE app_settings SET support_expiry_date = NULL WHERE id = 1;
-- أو تاريخ مستقبلي:
UPDATE app_settings SET support_expiry_date = '2027-12-31' WHERE id = 1;
```

---

## السيناريو 3: التحقق من إشعار تسجيل الدخول

1. تأكد أن `update-status.json` يحتوي على `"hasUpdate": true`
   - أو شغّل `checkForUpdate` مع إصدار جديد متاح
2. افتح `https://localhost:3000`
3. **النتيجة المتوقعة**: يظهر شريط إشعار "تحديث جديد متاح — الإصدار X.X.XX"

---

## السيناريو 4: التحقق من شريط التقدم أثناء التحميل

1. تأكد من وجود إصدار جديد ودعم فني ساري
2. اضغط "تحديث الآن"
3. راقب شريط التقدم في الإعدادات
4. **النتيجة المتوقعة**:
   - تتقدم الخطوات: backup → downloading → verify → replace
   - شريط التقدم يتحرك من 0% إلى 100%
   - تظهر أرقام الـ bytes المُنزَّلة (مثل: 23.5 MB / 50 MB)

---

## التحقق من Rollback

لاختبار الـ rollback:
1. في بيئة تطوير: قم بتعطيل صلاحية الكتابة على ملف `laundry-app.exe` قبل الاستبدال
2. ابدأ التحديث
3. **النتيجة المتوقعة**: البرنامج يُعاد تشغيله بالإصدار القديم، ويظهر `"lastUpdateResult": { "status": "rollback" }` في `update-status.json`

---

## ملفات التحقق

- `{DATA_ROOT}/data/update-status.json` — حالة آخر فحص ونتيجة آخر تحديث
- `{DATA_ROOT}/data/update-log.txt` — سجل تفصيلي بطوابع زمنية
