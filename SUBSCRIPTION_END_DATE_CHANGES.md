# إضافة حقل تاريخ انتهاء الباقة للاشتراكات

## الملخص
تم إضافة حقل "تاريخ انتهاء الباقة" (end_date) للاشتراكات، بحيث يكون اختيارياً:
- إذا تم تحديد تاريخ انتهاء، تنتهي الباقة في ذلك التاريخ
- إذا تُرك فارغاً، تكون الباقة مفتوحة بدون تاريخ انتهاء محدد

## التغييرات المطبقة

### 1. قاعدة البيانات (database/db.js)

#### إضافة عمود end_date لجدول customer_subscriptions
```sql
ALTER TABLE customer_subscriptions 
ADD COLUMN end_date DATE DEFAULT NULL AFTER current_package_id
```

#### دالة Migration جديدة
- `ensureSubscriptionEndDateColumn()`: تضيف العمود للجداول الموجودة تلقائياً

#### تعديل دالة createSubscription
- إضافة معامل `endDate` 
- حفظ قيمة `end_date` عند إنشاء اشتراك جديد

#### تعديل دالة updateActiveSubscriptionPeriod
- إضافة معامل `endDate`
- تحديث قيمة `end_date` في جدول customer_subscriptions عند التعديل

#### تعديل دالة subscriptionListSelectSql
- إضافة حقل `cs.end_date` في استعلام جلب الاشتراكات

### 2. واجهة المستخدم (screens/subscriptions/)

#### HTML (subscriptions.html)

**نموذج إنشاء اشتراك جديد (modalNewSub):**
- إضافة حقل `newSubEnd` لإدخال تاريخ الانتهاء
- استخدام مفتاح ترجمة `subscriptions-label-end-date`
- نص توضيحي: "اتركه فارغاً للباقة المفتوحة"

**نموذج تعديل الاشتراك (modalEditSub):**
- إضافة حقل `editSubEndDate` لتعديل تاريخ الانتهاء
- استخدام مفتاح ترجمة `subscriptions-label-end-date`
- نص توضيحي: "اتركه فارغاً للباقة المفتوحة"

**تنسيق الأرقام:**
- إضافة `lang="en"` لجميع حقول التاريخ والأرقام
- إضافة CSS لجعل الأرقام تظهر بالإنجليزية:
  ```css
  input[type="date"],input[type="number"]{
    font-family:'Segoe UI',Tahoma,sans-serif;
    direction:ltr;
    text-align:right
  }
  ```

#### JavaScript (subscriptions.js)

**عند فتح نموذج اشتراك جديد:**
- تنظيف حقل `newSubEnd` عند فتح النموذج

**عند حفظ اشتراك جديد:**
- قراءة قيمة `newSubEnd`
- إرسالها كـ `endDate` في الطلب

**عند فتح نموذج تعديل اشتراك:**
- تحميل قيمة `end_date` من البيانات إلى حقل `editSubEndDate`

**عند حفظ تعديل اشتراك:**
- قراءة قيمة `editSubEndDate`
- إرسالها كـ `endDate` في الطلب

### 3. الترجمة (assets/i18n.js)

**مفاتيح الترجمة العربية:**
- `subscriptions-label-end-date`: "تاريخ انتهاء الباقة"
- `subscriptions-label-end-hint`: "اتركه فارغاً للباقة المفتوحة"

**مفاتيح الترجمة الإنجليزية:**
- `subscriptions-label-end-date`: "Package end date"
- `subscriptions-label-end-hint`: "Leave empty for open-ended package"

## كيفية الاستخدام

### إنشاء اشتراك جديد
1. افتح نموذج "اشتراك جديد"
2. اختر العميل والباقة
3. حدد تاريخ البداية (اختياري)
4. **حدد تاريخ انتهاء الباقة (اختياري)**:
   - إذا حددت تاريخ: ستنتهي الباقة في هذا التاريخ
   - إذا تركته فارغاً: الباقة مفتوحة بدون تاريخ انتهاء

### تعديل اشتراك موجود
1. افتح نموذج "تعديل الاشتراك"
2. عدّل تاريخ بداية/نهاية الفترة
3. **عدّل تاريخ انتهاء الباقة (اختياري)**:
   - حدد تاريخ جديد لتحديد موعد انتهاء
   - امسح التاريخ لجعل الباقة مفتوحة

## ملاحظات تقنية

- الحقل اختياري (nullable) في قاعدة البيانات
- يتم التحقق من القيمة في الـ backend قبل الحفظ
- إذا كانت القيمة فارغة أو null، يتم حفظها كـ NULL في قاعدة البيانات
- التاريخ يُحفظ بصيغة DATE (YYYY-MM-DD)
- Migration تلقائي يضيف العمود للجداول الموجودة عند بدء التطبيق
- الأرقام تظهر بالإنجليزية في جميع حقول الإدخال
